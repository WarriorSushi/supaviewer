## Design Context

### Users
Cinephiles and creators discovering AI-native films. Users are browsing a curated catalog of serial-numbered titles, watching premiere rooms, following creators, and exploring agent-curated collections. Context: lean-back discovery (like browsing a boutique streaming service), lean-forward engagement (premiere lounges, comments, submissions). The job: find remarkable AI films, connect with the directors making them, and participate in live screening events.

### Brand Personality
**Cinematic. Authoritative. Warm.** Supaviewer is a serious film platform with editorial taste — not a social feed, not a tech demo. Think Criterion Collection meets a modern screening room. The voice is confident and knowledgeable but never cold. Speaks like a trusted film programmer, not a marketing bot.

### Aesthetic Direction
**Editorial Cinema** — Magazine-quality typography with cinematic imagery. Dark-first with warm amber/copper accents that feel like projector light on film stock. Large type, generous whitespace, dramatic image crops. The interface should feel like opening a beautifully designed film journal, not scrolling a video grid.

- **References**: Criterion Channel (editorial authority), MUBI (curated taste), A24 (bold type + imagery), Letterboxd (social warmth)
- **Anti-references**: YouTube (chaotic grid), Netflix (algorithmic sameness), generic shadcn dashboards
- **Theme**: Dark-first. Warm neutrals tinted toward amber/sepia. OKLCH color space for perceptual accuracy.

### Design Principles
1. **Cinema over chrome** — Let film imagery dominate. UI elements recede; content commands the frame. No decorative borders, no unnecessary containers.
2. **Editorial typography** — Fraunces display for headings creates a literary, curated feel. Plus Jakarta Sans for body. Type hierarchy does the heavy lifting — not cards, not borders.
3. **Purposeful warmth** — Amber/copper accent (not cold blue/teal). Tinted neutrals, not pure gray. The palette should feel like warm light, not a tech product.
4. **Breathe and reveal** — Generous spacing between sections. Progressive disclosure over information dumps. Every page has a clear focal point and a reading rhythm.
5. **Earned motion** — Subtle, cinematic transitions. Fade-ins on scroll, smooth page transitions. No bouncing, no elastic — things move like a slow dolly shot, not a UI demo.

### Typography
- **Display**: Fraunces (serif, optical, 500-700 weight) — for hero headings, section titles, film titles on detail pages
- **Body**: Plus Jakarta Sans (geometric sans, 400-600) — for all body copy, navigation, labels, metadata
- **Mono**: IBM Plex Mono (400-500) — for serial numbers, technical metadata, code-like data
- **Scale**: Fluid with clamp(). Hero: 3.2rem-5rem. Section title: 1.6rem-2.4rem. Body: 0.94rem-1rem.

### Color Palette (OKLCH)
- **Background**: oklch(0.09 0.008 65) (near-black with warm sepia tint)
- **Surface**: oklch(0.13 0.008 65) (elevated dark with warmth)
- **Surface Elevated**: oklch(0.16 0.01 65) (cards, panels)
- **Foreground**: oklch(0.93 0.008 85) (warm off-white, not pure white)
- **Muted**: oklch(0.55 0.01 65) (secondary text, warm gray)
- **Accent**: oklch(0.72 0.14 55) (warm amber/copper — projector light)
- **Accent Strong**: oklch(0.65 0.16 50) (deep copper for hover/active)
- **Destructive**: oklch(0.60 0.20 25) (warm red, not neon)
- **Border**: oklch(1 0 0 / 7%) (subtle white borders)

### Radius
- Cards/panels: rounded-2xl (1rem)
- Buttons: rounded-xl (0.75rem)
- Chips/pills: rounded-full
- Film thumbnails: rounded-xl with overflow-hidden
- No rounded-3xl or larger — keep it refined, not bubbly

### Components
- shadcn/ui (New York style) + Radix primitives
- Lucide icons (stroke width 1.6, not chunky)
- Framer Motion for page transitions and scroll reveals (lazy-loaded)
- Custom `sv-*` CSS classes for brand-specific styling

### Spacing Philosophy
- Sections separated by 3-4rem (not uniform — vary for rhythm)
- Hero sections get generous 5-6rem bottom padding
- Card grids: 1.25rem gap (tight, editorial feel)
- Text blocks: tight leading (1.3-1.4 for display, 1.6 for body)
- Max content width: 72rem for reading, 96rem for grids
